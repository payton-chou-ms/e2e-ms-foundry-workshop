# 參與者執行與驗證

當 Azure 和 Fabric 環境已經為你準備好時，請走這條路徑。

## 這條路徑適合誰

- 已拿到現成環境的學員
- 想先跑通範例情境的學員
- 想先驗證代理程式行為，再回頭看部署細節的學員

## 這條路徑涵蓋的內容

1. 使用指定身分登入。
2. 開啟 repo 並設定本機環境。
3. 驗證範例情境。
4. 測試代理程式。

## 開始前你應該已拿到的資訊

- 存取 Azure 訂用帳戶或已準備好的資源的權限。
- 存取 Fabric 工作區的權限。
- 正確的 `FABRIC_WORKSPACE_ID`。
- 這套環境是否已完成範例建置。

## 建議執行順序

### 1. 確認登入與工作區存取

確認你可以驗證到正確的 Azure 租用戶，並開啟目標 Fabric 工作區。

如果你無法存取工作區或 Azure 資源，請先停下來，確認權限後再繼續。

### 2. 設定本機環境

請依照 [設定開發環境](03-configure.md) 操作。

最低限度，你的 `.env` 應包含共用的 `FABRIC_WORKSPACE_ID` 和預期的 `DATA_FOLDER`。

### 3. 執行範例情境

請依照 [建置解決方案](04-run-scenario.md) 操作。

選擇正確的起始點：

- 如果環境已部署完成但尚未執行建置，請依照文件完成範例建置流程。
- 如果環境已完成範例建置，你通常可以直接跳到測試步驟。

### 4. 驗證代理程式行為

執行範例問題，確認你可以測試：

- 結構化資料問題
- 文件問題
- 同時使用兩個資料來源的組合式問題

## 這條路徑不涵蓋的內容

- 從零開始佈建 Azure 資源
- 從零開始建立 Fabric 工作區
- 授予其他使用者 Azure RBAC
- 將環境分享給額外的參與者

這些工作請改走 [管理員部署與分享](00-admin-deploy-share.md)。

## 驗證後的下一步

範例情境正常運作後，請繼續至 [為你的使用案例自訂](../02-customize/index.md)。
